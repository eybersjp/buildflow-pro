# Contributing to BuildFlow Pro

First off, thank you for considering contributing to BuildFlow Pro! It's people like you that make BuildFlow Pro such a great tool for building production-ready apps with AI.

## How to Contribute

### 1. Reporting Bugs
If you find a bug or issue, please open an issue on the GitHub repository. Describe the bug in detail, including steps to reproduce it, the expected behavior, and the actual behavior.

### 2. Suggesting Enhancements
We welcome suggestions for new workflows, skills, templates, or general improvements. Please open an issue to discuss your idea before submitting a pull request to ensure it aligns with the project's goals.

### 3. Submitting Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bug fix (`git checkout -b feature/your-feature-name` or `git checkout -b fix/your-bug-fix`).
3. Make your changes, ensuring they adhere to the project's coding standards.
4. If you are adding a new skill or workflow, be sure to document it well in the corresponding `.md` files.
5. Commit your changes (`git commit -m 'feat: Add some feature'`).
6. Push to the branch (`git push origin feature/your-feature-name`).
7. Open a Pull Request and describe your changes.

## Development Setup

To work on BuildFlow Pro locally:
1. Clone your fork: `git clone https://github.com/YOUR_USERNAME/buildflow-pro.git`
2. Make your changes in the `kit/` directory or `cli/index.js`.
3. To test the CLI locally, you can run `node cli/index.js init` inside a test directory or use `npm link`.

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. We expect all contributors to be respectful and professional.
